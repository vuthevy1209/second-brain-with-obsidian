# Recap

To be effective with Claude Code, follow the Explore, Plan, Code, and Commit workflow:

- Explore gives Claude the relevant context it needs for your project.
- Plan crates a plan of action that Claude uses to measure success.
- Code is the back and forth between you and Claude before settling on the final outcome.
- Commit helps you review and push your code so you can start on your next feature.

# The Explore → Plan → Code → Commit Workflow

> [!SUMMARIZE]
> - If you take on thing away from this course, let it be this workflow: Explore, Plan, Code, and Commit.
> - Without it, most people jump straight to asking Claude to write code - which means more course-correcting later on.

### Explore and Plan

The fastest way to handle these first two steps is with Plan Mode. In plan mode, Claude can't edit files - it just reads files to gather information about how it will tackle the implementation.

Use `Shift + Tab` to switch back and forth between Plan mode and Auto-Accept vs. Approval mode

![[4.Workflow in Claude Code-1777892992026.webp]]

**Example with a prompt:**
``` plain text
I need to add WebP conversion to our image upload pipeline. Figure out where in the pipeline it should happen, whether we need new dependencies, and how to approach it.
```

- Claude will read relevant files, run some web searches, and give you a plan of action. Review it and decide if it meets your criteria. If not, ask it to revise specific areas. 

![[4.Workflow in Claude Code-1777893007811.webp]]

This is the best place to course-correct because it's before any code is written.

### Code

> [!summary] Quick tip
> If you find Claude keeps running into the same issues, ask it to save the solution to its CLAUDE.md file

A few tips to make the coding phase smoother:

- **Define a success criteria:** For Claude to be confident in its results, it needs to be clear on what "correct" looks like. Make this explicit when writing your plan.
- **Add tools:** Tools that help Claude complete its goals remove a lot of back and forth. 

	Example, if you're building web UIs, install the Claude in Chrome extension so Claude Code can control a browser tab and test UI directly.

  
![[4.Workflow in Claude Code-1777879978727.webp]]

- **Include a test result:** Give Claude a test suite it can continuously validate against.

### Commit

Once you've tested the changes yourself and are happy the results, it's time to push your code. Before commit, run a subagent code reviewer to look at your work.

![[4.Workflow in Claude Code-1777885207860.webp]]

Then get Claude to generate a commit message in your style. Rinse and repeat.
