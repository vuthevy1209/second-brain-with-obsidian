# Context Management

Claude Code uses a 200k token context window with intelligent compaction and persistent memory to maintain conversation state across sessions.

![[5.Context Management-1777887165919.webp]]

## Context Window

The context window is the amount of text Claude can "see" at once:
- **200,000 tokens** (~150,000 words)
- Includes system prompt, conversation history, and file contents
- When full, Claude uses compaction to summarize older content

![[5.Context Management-1777887057869.webp]]

| Category      | Description                                  |
| ------------- | -------------------------------------------- |
| System prompt | Core instructions (Claude.md, skills, tools) |
| System tools  | Available tool definitions                   |
| Memory files  | CLAUDE.md and memory/ directory              |
| Skills        | Loaded skill definitions                     |
| Messages      | Actual conversation history                  |
| Free space    | Remaining available tokens                   |

## Auto-Compact

When context approaches the limit, Claude automatically compacts older messages:

- **Compaction buffer**: ~33k tokens reserved for compaction
- **Compaction trigger**: When messages + tools approach ~167k tokens
- **What happens**: Older conversation turns are summarized into a compact narrative
- **Preserved**: Recent messages, important decisions, and active file state remain uncompressed

### Compaction Behavior
```
Before: Full conversation history with all details
After:  Summary of earlier discussion + recent messages intact
```

![[5.Context Management-1777887223812.webp]]

If you want to completely start from scratch with no memory of the previous session, run `/clear`. This removes everything.
