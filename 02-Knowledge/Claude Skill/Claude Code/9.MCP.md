## Recap

MCP (Model Context Protocol) is an open standard that enables Claude Code to connect with external tools and data sources. It uses a client-server architecture where the host app runs an MCP client that communicates with MCP servers. These servers expose capabilities through tools, resources, and prompts. MCP servers add tool definitions to your context window even when not actively used, so manage them wisely using `/mcp` command or restrict tools via CLAUDE.md.

# What is MCP

<mark style="background:rgba(3, 135, 102, 0.2)">Model Context Protocol (MCP) is an open standard that lets Claude Code connect to external tools and data source</mark>

- When you ask a question, Claude automatically understands when it should use those tools to better handle your query.
- A lot of your context lives outside your codebase such as in database, productivity apps, or public repositories.
- MCP bridges that gap.

![[9.MCP-1777950606579.webp|499x588]]

**Host** represents any AI app (Claude desktop, Cursor) that provides an environment for AI interactions, accesses tools and data, and runs the MCP Client.

**MCP Client** operates within the host to enable communication with MCP servers.

![[9.MCP-1777950726463.webp|377x180]]

Finally, the **MCP Server** exposes specific capabilities and provides access to data like:

![[9.MCP-1777950755236.webp|307x294]]
- **Tools:** Enable LLMs to perform actions through your server.
- **Resources:** Expose data and content from your servers to LLMs.
- **Prompts:** Create reusable prompt templates and workflows.

# How it works?

> [!abstract] Flow
> Your Agent prompt → LLM call (your tokens) → LLM decides to call MCP tool → MCP server executes → result returned to your agent's context → LLM continues (your tokens again)

- **Your Agent (the MCP client)** initiates the connection and sends requests to the MCP server.
- **The MCP server** is just a tool provider — it receives calls from your agent, executes them (e.g., read a file, query a DB, call an API), and returns results.
- **The tokens consumed** come from your agent's LLM call, because the MCP server's responses are fed back into your agent's context window as tool results.

# Adding an MCP Server

You can add MCP servers with `claude mcp add` command. 
There are two main types:

- <mark style="background:rgba(3, 135, 102, 0.2)">HTTP servers</mark> are for remote services.
- <mark style="background:rgba(3, 135, 102, 0.2)">Stdio servers</mark> are for local processes that run on your machine

Scope Servers: MCP servers can be scoped in three ways. If you don't specify a scope when adding, it defaults to **Local**.

1. **Local:** Only available in the current project, just for you. Stored in `~/.claude/settings.json` inside project field.
	![[9.MCP-1777958163166.webp]]

2. **User:** Available across all your projects, just for you. Stored in `~/.claude/settings.json`.
3. **Project:** Uses a `.mcp.json` file that you check into version control so anyone on the codebase gets the exact same servers automatically.

> [!cite] Example
> **Example: Connect with Notion MCP (http)** with user scope.
> Read more: [Connecting to Notion MCP](https://developers.notion.com/guides/mcp/get-started-with-mcp#using-scope-flag-for-different-installation-scopes)

```
claude mcp add --transport http notion https://mcp.notion.com/mcp
```

![[9.MCP-1777956832469.webp]]

# Context Costs

![[9.MCP-1777958287814.webp]]

<mark style="background:rgba(136, 49, 204, 0.2)">MCP servers add tool definitions to your context window - even when you're not actively using them.</mark>

If you have a lot of servers configured => This eats into your available context.

Use `/mcp` command and select on MCP server to see what's connected and disable anything you're not actively using.

![[9.MCP-1777958917085.webp]]

**Example:** I uses Notion MCP to read and write docs, So I will not use some tools like creating or getting comment => I have disable it, but currently: Claude Code does not have a way to individually disable each tool => Go into config and delete it manually, <mark style="background:#ff4d4f">but it's not recommended</mark>.

![[9.MCP-1777958716439.webp]]

Use allowedTools in CLAUDE.md to restrict actions.

```
## MCP Tool Restrictions

Only use these Notion tools:
- notion-search
- notion-fetch
- notion-create-pages
- notion-get-comments

Do NOT use: notion-duplicate-page, notion-update-data-source, notion-create-database
```
