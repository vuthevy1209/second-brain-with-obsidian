# Key takeaways

- A skill is a directory containing a `SKILL.md` file with metadata (name, description) in frontmatter and instruction below.
- Claude loads only skill names and description at startup, then matches incoming request against those descriptions using semantic matching.
- <mark style="background:#d3f8b6">You get a confirmation prompt before Claude loads the full skill content into context.</mark>
- Priority for name conflicts:<mark style="background:#40a9ff"> Enterprise → Personal → Project → Plugins</mark>
- To update a skill, edit its `SKILL.md`. To remove one, delete its directory. **Always restart Claude Code** for changes to take effect

# Create Skill

We'll build a project skill level that teaches Claude how to write <mark style="background:#affad1">Recap of a note</mark>.

The name identifies your skill. The description tells Claude when to use it

```
mkdir .claude/skills/write-recap/SKILL.md
```

```
---
name: write-recap
description: Writes a recap note with a quick summary and detailed bullet points with bold keywords.
---

When asked to write a recap or summary, you MUST follow this exact format:

**Quick Summary**
- [Brief summary point 1]
- [Brief summary point 2]
...

- [Detailed description for point 1]
- [Detailed description for point 2]
...

## Formatting Rules:
1. Always start with the exact text "**Quick Summary**" as a heading or bold text.
2. Follow with a brief bulleted list (`- `) summarizing the main points.
3. After the quick summary, provide a detailed description for each point, also using bullet points (`- `).
4. In the detailed descriptions, you MUST **bold** the most **important information/keywords** in every bullet point.

## Example Output:
**Quick Summary**
- Completed UI design
- Optimized database

- Completed the **UI design** for the homepage, focusing on **user experience (UX)** and **page load time**.
- Optimized the **database** by adding **indexes** to frequently queried tables, reducing **processing time by 70%**.

```

# How Skill Matching Works

When Claude Code starts, it scans for locations for skills but <mark style="background:#d3f8b6">only loads the name and description - not full content.</mark>

When you send a request, Claude compares your message against the description of all available skills. Once a match is found, Claude asks you to confirm loading the skill. After you confirm, Claude reads the complete `SKILL.md` file and follows its instructions.

# Skill Priority

If you clone a repository that has a skill with the same name as one of your personal skills, There's a clear priority order:

1. **Enterprise** — managed settings, highest priority
2. **Personal** — your home directory (`~/.claude/skills`)
3. **Project** — the `.claude/skills` directory inside a repository
4. **Plugins** — installed plugins, lowest priority

This lets organizations enforce standards through enterprise skills while still allowing individual customization.

If your company has an enterprise "code-review" skill and you create a personal "code-review" skill with the same name, the enterprise version takes precedence.

# Updating and Removing Skills

- To update a skill, edit its `SKILL.md` file. 
- To remove one, delete its directory. Restart Claude Code after any changes for them to take effect.
